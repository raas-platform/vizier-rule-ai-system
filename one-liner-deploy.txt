# 🚀 VizierAI 원라이너 배포 명령어
# 서버 접속 후 아래 명령어를 복사하여 붙여넣기하세요

## 1. 전체 자동 배포 (추천)
```bash
curl -fsSL https://raw.githubusercontent.com/yeonjae-work/vizier-rule-ai-system/main/quick-deploy.sh | bash
```

## 2. 수동 단계별 배포
```bash
# 프로젝트 디렉토리 생성 및 이동
sudo mkdir -p /opt/vizierai && sudo chown $USER:$USER /opt/vizierai && cd /opt/vizierai

# 리포지토리 클론 (최초 1회)
git clone https://github.com/yeonjae-work/vizier-rule-ai-system.git . || git pull origin main

# 기존 서비스 중지 및 정리
docker-compose down 2>/dev/null || true && docker system prune -f 2>/dev/null || true

# 환경 설정
cp env.template .env && echo -e "ENVIRONMENT=production\nDEBUG=false\nLOG_LEVEL=info\nPORT=8000\nWORKERS=4\nALLOWED_ORIGINS=*" >> .env

# 서비스 시작
docker-compose up -d --build && sleep 30 && curl -f http://localhost:8000/health && echo -e "\n✅ 배포 완료!"
```

## 3. 긴급 복구 원라이너
```bash
cd /opt/vizierai && docker-compose down && docker system prune -f && git reset --hard origin/main && git pull && docker-compose up -d --build --force-recreate
```

## 4. 서비스 상태 확인
```bash
docker ps && echo "---" && curl -s http://localhost:8000/health && echo -e "\n---" && docker-compose logs --tail=20
```

## 5. API 키 설정 (배포 후 필수)
```bash
# .env 파일 편집
nano .env
# 또는
vim .env

# 다음 라인들을 실제 API 키로 변경:
# OPENAI_API_KEY=your_actual_openai_key
# ANTHROPIC_API_KEY=your_actual_anthropic_key  
# GOOGLE_API_KEY=your_actual_google_key

# 서비스 재시작
docker-compose restart
```

## 6. 배포 검증
```bash
echo "=== 배포 검증 ===" && \
echo "서비스 상태:" && docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" && \
echo -e "\n헬스체크:" && curl -s http://localhost:8000/health && \
echo -e "\n\n외부 접속 URL:" && echo "http://$(curl -s ifconfig.me):8000" && \
echo "문서: http://$(curl -s ifconfig.me):8000/docs"
```

## 사용법:
1. AWS 콘솔에서 EC2 Instance Connect로 서버 접속
2. 위의 명령어 중 하나를 복사하여 붙여넣기
3. API 키 설정 후 서비스 재시작
4. 브라우저에서 접속 확인
